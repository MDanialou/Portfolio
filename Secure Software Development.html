Secure Software Development

Module Objective

This module introduces the underpinning concepts and skills of Secure Software Development. It is designed to help students acquire a critical understanding of the concept of abstraction in programming, develop an understanding of the basic principles of secure development methodologies, explore how to undertake analysis, program design, software construction and testing required for software development, demonstrate an understanding of the basic principles of architecture, as well as traditional and contemporary Software Development Life Cycle (SDLC) models, such as TOGAF and Agile, the ability to put into practice the techniques learned in a team environment, demonstrating how to deal with conflicts and how to make compromises, and be able to critically evaluate the effectiveness of the approaches, utilise the opportunity to reflect on and evaluate your own personal development.

Module Outcomes

Identify and manage security risks as part of a software development project.

Critically analyse development problems and determine appropriate methodologies, tools and techniques (including program design and development) to solve them.

Design and develop/adapt computer programs and to produce a solution that meets the design brief and critically evaluate solutions that are produced.

Systematically develop and implement the skills required to be effective member of a development team in a virtual professional environment, adopting real-life perspectives on team roles and organisation.

My initial post was focused on Security logging and monitoring is a vital part of an organization's network management and system administration. Logs indicate current status of a system and contain information that refers to different security events, which occur within the system. Logs are used for different purposes, such as recording user activities, track authentication attempts, and other security events (Söderström and Moradian, 2013). As one of the OWASP Top 10 in 2021 (OWASP, 2021), Security logging and monitoring failures impact the ability of an organization to detect and combat breaches due to the lack of visibility into system and application operations. Organizations that do not value the importance of logging and monitoring may have to face issues in case of a breach or incident due to absence of records and evidence, or lax data management practices, which may also lead to legal, contractual or regulatory non-compliance (Vasant and Saloni, 2017) The following UML use case diagram describes the high level functions and scope of a system. Unit 1: Introduction to Secure Software Development

Exploration of the waterfall and agile approaches to software development, with a focus on the implications of developing secure software using each. Acquaintance with the Unified Modelling Language and how it can be used to support software development. Familiarity with the standards which are used by industry to develop secure software. Appreciation of the importance of developing a risk-aware culture within an organisation.

Unit 2: UML Modelling to Support Secure System Planning

Awareness of the opportunities for integrating software security development practices at each stage of the agile SDLC. Understanding of the range of ways in which software security can be problematic.

Unit 3: Introduction to Programming Languages

Review of the history and concepts of programming languages. Investigation of language concepts using Python as an example. Description of best practices and methods to overcome common security issues.

Unit 4: Introduction to Testing

Exploration of the range of questions to ask when developing a test plan. Become familiar with two industry software testing standards. Examine a range of testing techniques and appreciate the relevance of each in different testing scenarios. Recognise the tools and frameworks which are available to support and automate the Python testing process. Reflect on the utility of an ontology to support a distributed architecture.

Unit 5: Future trends in Secure Software Development

Discussion on future trends in the areas of secure design, development, programming languages and operating systems. Explanation of what is meant by Fog Computing, the Internet of Things and Cyber Physical Systems. Exploration systems and solutions that can be investigated as possible research topics or fields.

Unit 6: The Security of Programming Languages Debate

Review of the security capabilities of a selection of programming languages, which include F#, Python, Rust, and Swift. Engagament in a debate about the above languages. Exploration of some of the advances needed to support secure programming.
